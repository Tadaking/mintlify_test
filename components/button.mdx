## 概要

{/* TODO: component description がないため構造から推測。_guidelines フレーム追加後に更新 */}

Button は、ユーザーがアクションを実行するためのコンポーネント。フォームの送信、ダイアログの確認、操作のトリガーなど、明確なアクションを促す場面で使用する。

## 類似要素との使い分け

{/* TODO: _guidelines フレームを追加後に更新 */}

| 判断基準 | 使用する要素 |
|---|---|
| アクションを実行する（保存、削除、送信など） | Button |
| 別のページ・URLに移動する | Link |
| 選択肢のオン/オフを切り替える | Toggle / Checkbox |

## バリアント

### variant

#### Primary

画面内で最も重要なアクションに使う。視覚的に最も目立つバリアント。

✅ いつ使う
- フォームの送信ボタン（「変更を保存」「アカウントを作成」）
- ダイアログの確定アクション
- ページ内で1つだけの主要アクション

❌ いつ使わない
- 同じ画面に複数の Primary を並べない
- 補助的・二次的なアクション（Secondary や Ghost を使う）

{/* illustration-needed: Do/Don't（_guidelines 追加後に更新） */}

#### Secondary

Primary に次ぐ重要度のアクション。ボーダーによって存在感を保ちつつ、Primary を引き立てる。

✅ いつ使う
- Primary と並べる補助アクション（「キャンセル」「戻る」）
- 単独で使う中程度の重要度のアクション

❌ いつ使わない
- 画面の主要アクション（Primary を使う）
- 目立たせたくない補助操作（Ghost を使う）

#### Neutral

控えめな見た目で、コンテンツを邪魔しないアクション。背景とボーダーで最低限の存在感を示す。

✅ いつ使う
- ツールバーやカード内の操作ボタン
- 一覧画面の各行に付くアクション

❌ いつ使わない
- ユーザーに積極的に押してほしいアクション（Primary / Secondary を使う）

#### Ghost

背景を持たず、テキストとアイコンだけで構成される最も控えめなバリアント。

✅ いつ使う
- ナビゲーション内の操作
- 密集したUI内で視覚的なノイズを減らしたい場面
- 「もっと見る」「閉じる」などの補助的な操作

❌ いつ使わない
- ユーザーが見落としてはいけない重要なアクション

### intent

#### Neutral

デフォルトの意図。一般的なアクションに使う。

#### Danger

削除や取り消しなど、破壊的・不可逆なアクションに使う。variant が Primary の場合は赤い背景、Secondary の場合は赤いテキストとボーダーになる。

### size

| サイズ | 用途 |
|---|---|
| xl（56px） | ランディングページやヒーローセクションなど、大きく目立たせたい場面 |
| lg（48px） | フォームの送信ボタンなど、標準的な主要アクション |
| md（36px） | カード内やツールバーなど、標準的なUI |
| sm（32px） | テーブルの行内アクションなど、省スペースが求められる場面 |

### その他のプロパティ

| プロパティ | 説明 |
|---|---|
| full-radius | `true` にすると角丸が最大になり、ピル型のボタンになる |
| hasLeftIcon / hasRightIcon | アイコンの表示・非表示を切り替える |
| isPending | 処理中状態。ローディングインジケーターを表示する |

## 組み合わせパターン

{/* illustration-needed: Do/Don't（_guidelines 追加後に更新） */}

### 推奨される組み合わせ

- **Primary + Secondary**: 主要アクションと補助アクションの基本パターン
- **Primary + Ghost**: 主要アクションと「キャンセル」のようなシンプルな組み合わせ
- **Secondary + Ghost**: Primary がページ内の別の場所にある場合

### 避けるべき組み合わせ

- **Primary + Primary**: 視覚的な優先順位が崩れる
- **同じバリアントを3つ以上並べる**: 選択肢が多い場合はドロップダウンやメニューを検討する

### 3つ以上のアクションが並ぶ場合

最も重要なアクションを Primary、次を Secondary、それ以外を Ghost にする。4つ以上になる場合は「その他」メニューへの集約を検討する。

## ラベル・コンテンツの書き方

- **動詞＋名詞**の形式で書く：「保存」→「変更を保存」、「削除」→「アカウントを削除」
- アクションの結果が明確に伝わる表現を使う
- 「クリック」「こちら」「押す」などの曖昧な表現は避ける
- Danger intent の場合は「削除する」「取り消す」など、破壊的であることが伝わる表現を使う
- ラベルは短く。目安として日本語で8文字以内、英語で3単語以内

## アクセシビリティ

### キーボード操作

| キー | 動作 |
|---|---|
| `Tab` | ボタンにフォーカスを移動する |
| `Enter` / `Space` | ボタンのアクションを実行する |

### スクリーンリーダー

| 属性 | 値 | 説明 |
|---|---|---|
| `role` | `button` | ネイティブ `<button>` 要素を使えば自動付与 |
| `aria-label` | — | アイコンのみのボタンに必須。ラベルが見えている場合は不要 |
| `aria-disabled` | `true` | disabled 状態で付与。`disabled` 属性と併用する |
| `aria-busy` | `true` | isPending 状態で付与 |

### コントラスト（APCA基準）

| バリアント | 組み合わせ | 基準 |
|---|---|---|
| Primary | `color/button/primary/text` on `color/button/primary/background/default` | Lc 60以上 |
| Secondary | `color/button/secondary/text` on `color/button/secondary/background/default` | Lc 60以上 |
| Neutral | `color/button/neutral/text/default` on `color/button/neutral/background/default` | Lc 60以上 |
| Ghost | `color/button/neutral/text/default` on 親要素の背景色 | Lc 60以上 |

## インタラクションステート

| ステート | 挙動 |
|---|---|
| Default | 静止状態 |
| Hover | 背景色が1段階濃くなる（ライトモード）/ 明るくなる（ダークモード） |
| Pressed | 背景色がさらに1段階濃くなる |
| Focused | フォーカスリングが表示される。背景色は Default のまま |
| Disabled | カラートークンは変更せず、`opacity: 0.4` を一律適用。クリック不可 |
| Pending | Disabled と同じ見た目 + ローディングインジケーターを表示 |
