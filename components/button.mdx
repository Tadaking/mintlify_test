## 概要

{/* TODO: _guidelines の overview テキストが空のため構造から推測。テキスト追加後に更新 */}

Button は、ユーザーがアクションを実行するためのコンポーネントです。フォームの送信、ダイアログの確認、操作のトリガーなど、明確なアクションを促す場面で使いましょう。

## 類似要素との使い分け

{/* TODO: _guidelines の related テキストが空。テキスト追加後に更新 */}

| 判断基準 | 使用する要素 |
|---|---|
| アクションを実行する（保存、削除、送信など） | Button |
| 別のページ・URLに移動する | Link |
| 選択肢のオン/オフを切り替える | Toggle / Checkbox |

## バリアント

### variant

#### Primary

画面内で最も目立たせたいアクションに適しています。視覚的な存在感が最も高いため、ユーザーの注意を自然に引きつけます。

{/* figma-frame: Pvlx7A6oDLlnPF8MQYFDcq/5079:7370 | Primary プレビュー */}
![Primary](/images/button-variant-primary-light.png)

✅ いつ使う
- フォームの送信ボタン（「変更を保存」「アカウントを作成」）
- ダイアログの確定アクション
- ページ内で1つだけのメインアクション
- まったく同じアクションをページ内に複数配置する場合は使用可能だが、Secondary も検討する

❌ いつ使わない
- 同じ画面に複数の Primary を並べない
- 補助的・二次的なアクション（Secondary や Ghost を使う）

#### Secondary

Primary に次ぐ重要度のアクションに向いています。tint カラーによって存在感を保ちつつ、Primary を引き立てる役割を担います。

{/* figma-frame: Pvlx7A6oDLlnPF8MQYFDcq/5079:7406 | Secondary プレビュー */}
![Secondary](/images/button-variant-secondary-light.png)

✅ いつ使う
- Primary と並べる補助アクション（「下書き保存」）
- 中程度の重要度のアクション（複数配置可能）

❌ いつ使わない
- 画面の主要アクション（Primary を使う）
- 目立たせたくない補助操作（Ghost を使う）

#### Neutral

迷ったらまずこのバリアントから検討しましょう。控えめな見た目でコンテンツを邪魔せず、幅広い場面で使えます。

{/* figma-frame: Pvlx7A6oDLlnPF8MQYFDcq/5079:7442 | Neutral プレビュー */}
![Neutral](/images/button-variant-neutral-light.png)

✅ いつ使う
- ツールバーやカード内の操作ボタン
- 一覧画面の各行に付くアクション
- Primary と並べる補助アクション（「キャンセル」「戻る」）

❌ いつ使わない
- ユーザーに積極的に押してほしいアクション（Primary / Secondary を使う）

#### Ghost

背景を持たず、テキストとアイコンだけで構成される最も控えめなバリアントです。UI が密集する場面で視覚的なノイズを抑えたいときに効果的です。

{/* figma-frame: Pvlx7A6oDLlnPF8MQYFDcq/5079:7478 | Ghost プレビュー */}
![Ghost](/images/button-variant-ghost-light.png)

✅ いつ使う
- ナビゲーション内の操作
- 密集したUI内で視覚的なノイズを減らしたい場面
- 「もっと見る」「閉じる」「キャンセル」などの補助的な操作

❌ いつ使わない
- ユーザーが見落としてはいけない重要なアクション

### intent

#### Neutral

デフォルトの意図です。基本的にはこちらを選びましょう。

#### Danger

削除や取り消しなど、破壊的・不可逆なアクションであることを視覚的に伝えます。variant が Primary の場合は赤い背景、Secondary の場合は赤い tint カラー、Neutral の場合は赤字のボタンになります。Ghost には Danger intent は存在しません。

### その他のプロパティ

| プロパティ | 説明 |
|---|---|
| hasLeftIcon / hasRightIcon | アイコンの表示・非表示を切り替える |
| isPending | 処理中状態。ローディングインジケーターを表示する |

## ラベル・コンテンツの書き方

- **動詞＋名詞**の形式で書きましょう：「保存」→「変更を保存」、「削除」→「アカウントを削除」
- アクションの結果が明確に伝わる表現が効果的です
- 「クリック」「こちら」「押す」などの曖昧な表現は避けましょう
- Danger intent の場合は「削除する」「取り消す」など、破壊的であることが伝わる表現を使いましょう
- ラベルは短く保ちましょう。目安として日本語で8文字以内、英語で3単語以内です

## アクセシビリティ

### キーボード操作

| キー | 動作 |
|---|---|
| `Tab` | ボタンにフォーカスを移動する |
| `Enter` / `Space` | ボタンのアクションを実行する |

### スクリーンリーダー

| 属性 | 値 | 説明 |
|---|---|---|
| `role` | `button` | ネイティブ `<button>` 要素を使えば自動付与 |
| `aria-label` | — | アイコンのみのボタンに必須。ラベルが見えている場合は不要 |
| `aria-disabled` | `true` | disabled 状態で付与。`disabled` 属性と併用する |
| `aria-busy` | `true` | isPending 状態で付与 |

### コントラスト（APCA基準）

| バリアント | 組み合わせ | 基準 |
|---|---|---|
| Primary | `color/button/primary/text` on `color/button/primary/background/default` | Lc 60以上 |
| Secondary | `color/button/secondary/text` on `color/button/secondary/background/default` | Lc 60以上 |
| Neutral | `color/button/neutral/text/default` on `color/button/neutral/background/default` | Lc 60以上 |
| Ghost | `color/button/neutral/text/default` on 親要素の背景色 | Lc 60以上 |

## Size

{/* figma-frame: Pvlx7A6oDLlnPF8MQYFDcq/5078:6442 | サイズ比較（sm〜xl） */}

![Button Size](/images/button-size-light.png)

| サイズ | 用途 |
|---|---|
| sm（28px） | テーブルの行内アクションなど、省スペースが求められる場面 |
| md（36px） | カード内やツールバーなど、標準的なUI |
| lg（48px） | フォームの送信ボタンなど、標準的な主要アクション |
| xl（56px） | ランディングページやヒーローセクションなど、大きく目立たせたい場面 |

### min-width

すべてのサイズに min-width が設定されています。ラベルが短くてもボタンが極端に小さくならず、タップ領域と視覚的な安定感を確保します。

| サイズ | min-width |
|---|---|
| xl | 132px |
| lg | 96px |
| md | 88px |
| sm | 56px |
