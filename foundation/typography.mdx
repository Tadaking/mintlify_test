export const ScaleSwitch = ({ children }) => {
  const [scale, setScale] = React.useState('standard');
  return (
    <ScaleCtx.Provider value={scale}>
      <div className="typescale-switcher">
        {['standard', 'comfortable'].map(s => (
          <button key={s} className={scale === s ? 'active' : ''} onClick={() => setScale(s)}>
            {s === 'standard' ? 'Standard' : 'Comfortable'}
          </button>
        ))}
      </div>
      {children}
    </ScaleCtx.Provider>
  );
};

export const BP = ({ name, std, cmf, ls }) => {
  const scale = React.useContext(ScaleCtx);
  const [sz, lhs] = scale === 'standard' ? std : cmf;
  const [mode, setMode] = React.useState('standard');
  const lh = lhs[mode];
  const lsVal = ls ? (scale === 'standard' ? ls.std : ls.cmf) : 0;
  const labels = { comfortable: 'Comfortable', standard: 'Standard', compact: 'Compact' };
  return (
    <div className="type-preview type-preview--wrap">
      <div style={{display: 'flex', alignItems: 'center', justifyContent: 'space-between', marginBottom: '8px'}}>
        <div className="type-preview-name" style={{marginBottom: 0}}>{name}</div>
        <select value={mode} onChange={e => setMode(e.target.value)} className="type-preview-select">
          {Object.entries(lhs).map(([k, v]) => (
            <option key={k} value={k}>{labels[k]}（行間 {v}px）</option>
          ))}
        </select>
      </div>
      <div className="type-preview-sample-jp" style={{fontSize: sz+'px', fontWeight: 400, lineHeight: lh+'px', letterSpacing: lsVal+'px'}}>{JP_BODY}</div>
      <div className="type-preview-font">Noto Sans JP</div>

      <div className="type-preview-font">Inter</div>
      <div className="type-preview-specs">サイズ: {sz}px / {toRem(sz)}　ウェイト: {WL[400]}　行間: {lh}px / {toRem(lh)}　字間: {lsVal}px</div>
    </div>
  );
};

    # Type face

    プラットフォームごとに推奨のフォントファミリーを定義しています。iOS / Android はシステム標準フォント、Web は日本語と英語でフォントを分けて設計します。

    |     | Web          | Android    | iOS        |
    | --- | ------------ | ---------- | ---------- |
    | 日本語 | Noto Sans JP | システム標準フォント | システム標準フォント |
    | 英数  | Inter        | システム標準フォント | システム標準フォント |

      Safari（WebKit）ではシステム標準フォントにフォールバックします。

    ## Safari（WebKit）でのフォールバック

    Safari では `-apple-system`（システム標準フォント）を優先して使います。これは、iOS アプリ内の文字と、アプリの WebView 内で表示される文字を同じ見た目にそろえるための設計です。

    アプリと WebView でフォントが違うと、同じ画面でも文字サイズや行間が少しずれて見えてしまいます。このズレをなくすために、Safari ではシステムフォントを優先する構成にしています。

    ### **既知のトレードオフ:**

    - macOS / iOS の Safari では、日本語フォントがやや小さく見えるように調整されています
    - iOS アプリと Web ではタイプスケールが異なるため、日本語フォントサイズが WebView 上で小さく表示される
    - デスクトップ環境では、Windows（Inter 適用）と macOS Safari（San Francisco 適用）で、日本語の見た目のフォントサイズが変わってしまう

    ### **CSS font-family スタック:**

    ```css
    font-family: -apple-system, var(--font-inter), var(--font-noto-sans-jp), sans-serif, "Apple Color Emoji", "Segoe UI Emoji", "Segoe UI Symbol", "Noto Color Emoji";
    ```

    Safari はシステムフォント優先、他ブラウザは Inter + Noto Sans JP。

    # Type scale

    iOS / Android / Web それぞれに専用のタイプスケールを定義しています。さらに、Web と Android では画面密度に応じて 2 種類のタイプスケールを用意し、自動・またはレイアウト設計で切り替えられるようになっています。

    - **iOS**: 単一のタイプスケール — ディスプレイ、タイトル、本文、ラベル、キャプションを段階的に定義
    - **Android**: 2 系統のタイプスケール — ドライバー向けなどの低密度レイアウト向けスケール / 高密度レイアウト向けスケール
    - **Web**: 2 系統のタイプスケール — デスクトップやモバイル向けなどの高密度レイアウト向けスケール / 大画面・低密度レイアウト向けスケール（🚧準備中）

    # Semantic styles

    このデザインシステムでは、用途にもとづいたセマンティックなタイポグラフィスタイルを定義しています。数字ベースの「H1 / H2 / 16px」ではなく、役割にもとづいてスタイルを選択します。

    | カテゴリ             | 役割         | 主な使用場面例                                         |
    | :--------------- | :--------- | :---------------------------------------------- |
    | Big Title        | 最大の見出し     | ヒーローエリア                                         |
    | Display          | 装飾的見出し     | ページタイトル（装飾的）、セクション見出し（装飾的）、Dashboard などでのスタッツなど |
    | Title            | 機能的見出し     | ページタイトル（機能的）、カード見出し、モーダルタイトル、セクション見出し（機能的）      |
    | Body             | 本文（標準）     | 説明文、フォーム内テキスト                                   |
    | Body-Comfortable | 本文（ゆったり）   | 記事、長文読み物                                        |
    | Body-Compact     | 本文（詰まった表示） | データテーブル、リスト内テキスト                                |
    | Label            | ラベル・操作テキスト | ボタン、フォームラベル、ナビゲーション                             |
    | Caption          | 補助テキスト     | タイムスタンプ、注釈                                      |

    セマンティックスタイルは、プラットフォームごとにスケールやサイズ値は異なりますが、役割と使い分けの考え方は共通です。

    # Semantic styles

    このデザインシステムでは、用途にもとづいたセマンティックなタイポグラフィスタイルを定義しています。数字ベースの「H1 / H2 / 16sp」ではなく、役割にもとづいてスタイルを選択します。

    | カテゴリ | 役割 | 主な使用場面例 |
    | :-- | :-- | :-- |
    | Display | 装飾的な大きな見出し | ヒーローエリア、ダッシュボードの大きな数値 |
    | Headline | ページ・セクションの見出し | ページタイトル、セクションヘッダー |
    | Title | カード・モーダルの見出し | カードタイトル、モーダルタイトル、リスト見出し |
    | Body | 本文 | 説明文、記事本文、リスト内テキスト |
    | Label | ラベル・補助テキスト | フォームラベル、タブラベル、キャプション |
    | Button Label | ボタン内のラベル | ボタン、FAB のテキスト |

    セマンティックスタイルは、プラットフォームごとにスケールやサイズ値は異なりますが、役割と使い分けの考え方は共通です。

    ## 命名規則

    `{role}-{size}`

    例: `body-lg`, `title-md`, `button-label-xs`

    <Note>
      各カテゴリには `weight`（通常）と `weight-emphasized`（強調）の2種類のウェイトが定義されています。強調表示時は `weight-emphasized`（Bold 700）に切り替えます。Button Label は常に Bold（700）で定義されています。
    </Note>

    # Design for Driving

    車載ディスプレイ向けのタイポグラフィは、Google の [Design for Driving](https://docs.partner.android.com/drivingux/foundations/design-principles?hl=ja) のガイドラインに準拠しています。運転中の一瞥で情報を取得できることが最優先されるため、UI の通常スケールよりも厳しい制約があります。

    For Driving モードは、このガイドラインに従って行間・字間・サイズを調整したモードです。ドライバー向けの画面にはこのモードを適用します。

    ## プライマリ / セカンダリでのサイズ使い分け

    Google のガイドラインでは、テキストの役割によって必要な最小サイズが明確に定義されています。

    | 役割 | 最小サイズ | 典型例 |
    | :-- | :-- | :-- |
    | **プライマリ** | **32dp** | 意思決定に必要な情報：曲のタイトル、連絡先名、ナビ指示、速度など |
    | **セカンダリ** | **24dp** | 補足情報：アーティスト名、コールタイプ（「モバイル」など）、ステータス |
    | サブテキスト | 24dp 未満も可 | 非重要情報のみ（脚注、タイムスタンプ等） |

    Garage の For Driving モードでは以下のようにサイズが割り当てられています。

    | カテゴリ | For Driving サイズ (dp) | 用途の目安 |
    | :-- | :-- | :-- |
    | display-lg / md / sm | 56 / 44 / 40 | プライマリ（十分に大きい） |
    | headline-lg / md | 36 / 32 | プライマリ |
    | headline-sm | 28 | ⚠️ プライマリ基準（32dp）未満。セカンダリ用途で使う |
    | title-lg / md | 24 / 24 | セカンダリ |
    | title-sm | 22 | サブテキスト |
    | body-lg | 24 | セカンダリ |
    | body-md / sm | 22 / 20 | サブテキスト |
    | label-lg / md / sm | 22 / 20 / 18 | サブテキスト |
    | button-label-lg / md | 36 / 32 | プライマリ操作に使用可 |
    | button-label-sm / xs | 28 / 24 | セカンダリ操作 |

    <Warning>
      意思決定に関わる最重要テキスト（速度、警告、ナビゲーション指示、選択肢のタイトル等）には **32dp 以上**を使いましょう。Garage では display / headline-lg / headline-md / button-label-lg / button-label-md が該当します。
    </Warning>

    ## テキスト長・コントラスト

    - ローマ字テキスト: 句読点・スペースを含めて **120 文字以内**
    - 日本語テキスト: ローマ字・かな・漢字あわせて **31 文字以内**
    - アイコン・テキスト・画像のコントラスト比: **4.5:1 以上**

    ## フォントの使い分け（32dp 閾値）

    Google のガイドラインでは、サイズによって使用するフォントを切り替えることが推奨されています。

    - **32dp 以上**: Google Sans（ディスプレイ向け。大きいサイズで映える）
    - **32dp 未満**: Roboto（小さいサイズでも高い可読性を維持）
    - 日本語など Google Sans / Roboto でカバーされない言語は **Noto Sans** を使います

    Garage では「システム標準フォント」を指定していますが、Android Auto 等の車載環境で実装する際は、この 32dp 閾値に沿ってフォントを切り替える運用を推奨します。Garage のスケールだと display-sm（For Driving で 40dp）以上は Google Sans、headline-sm（28dp）以下は Roboto が目安になります。

    ## ウェイトの使い分け

    - 強調には **Medium（500）を優先** し、ここぞというときに限って使います
    - **Bold（700）の多用は避けます**。車載ディスプレイでは Medium の方が可読性が高いとされています
    - Button Label は例外的に Bold を使用しています（タッチ領域を明確にするため）
